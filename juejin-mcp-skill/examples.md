# 掘金 MCP 使用示例

## 示例 1：发布技术文章

**场景**：用户想要发布一篇关于 Python 的技术文章

**对话示例**：

```
用户：帮我发布一篇掘金文章，标题是「Python 异步编程入门指南」，
内容如下：

# Python 异步编程入门指南

异步编程是现代 Python 开发中的重要技能...

[文章内容...]

分类选择后端，标签加上 Python、异步编程、 asyncio

AI：已为您创建草稿并发布成功！
文章链接：https://juejin.cn/post/7382918472656487462
```

**实际调用**：

```json
{
  "tool": "publish_article",
  "arguments": {
    "title": "Python 异步编程入门指南",
    "content": "# Python 异步编程入门指南\n\n异步编程是现代 Python 开发中的重要技能...",
    "categoryId": "6809637769959178254",
    "tagIds": ["6809640485200245000", "6809640485200245001", "6809640485200245002"]
  }
}
```

## 示例 3：更新已发布文章

**场景**：用户想给已发布的文章添加更新标记

**对话示例**：

```
用户：请帮我更新文章 7382918472656487462，
在末尾添加「更新于 2024-01-20」

AI：正在获取文章详情...

已更新文章，在末尾添加了更新标记。
文章链接：https://juejin.cn/post/7382918472656487462
```

**实际调用**：

```json
// 第一步：获取文章详情
{
  "tool": "get_article",
  "arguments": {
    "articleId": "7382918472656487462"
  }
}

// 第二步：更新文章
{
  "tool": "update_article",
  "arguments": {
    "articleId": "7382918472656487462",
    "content": "[原有内容]\n\n---\n\n更新于 2024-01-20"
  }
}
```

## 完整工作流示例

### 工作流：发布一篇完整的文章

```
1. 获取分类列表 → list_categories
2. 获取标签列表 → list_tags
3. 创建草稿 → create_draft
4. （可选）更新草稿 → update_draft
5. 发布文章 → publish_article
```

### 工作流：修改已发布文章

```
1. 获取文章详情 → get_article
2. 修改内容
3. 更新文章 → update_article
```
